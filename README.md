# معيار — Meyar Dashboard System

منظومة الرقابة المالية اللحظية المدعومة بالذكاء الاصطناعي — واجهة تنفيذية فاخرة بتصميم داكن (Dark / Glassmorphism) مبنية بالكامل بالعربية (RTL).

## ما الجديد (نسخة الاستعداد للجنة التحكيم)

- **نموذج المنع المتدرج على مستويين**: مستوى ١ (قواعد قطعية قابلة للتحقق آلياً) → منع آلي فوري. مستوى ٢ (حالات اجتهادية كالشبهة الشرعية أو مؤشرات غسل الأموال) → تعليق وإحالة صريحة لمراجعة بشرية (موظف امتثال أو الهيئة الشرعية)، بلا أي قرار نهائي آلي.
- **تبويب "الحدود والمسؤولية"**: إفصاح صريح عن حدود دقة النظام، من يتحمل المسؤولية في كل مستوى، حدود صلاحيات الخدمات المصرفية المفتوحة، ودور الهيئة الشرعية.
- **منهجية موثّقة لرقم خفض التكاليف** بدل رقم تسويقي بلا مصدر (متاحة عبر أيقونة معلومات بجانب النسبة).
- **مساعد تشريعات (Chatbot)** يجاوب حصراً من قاعدة معرفة محلية محدودة لتعاميم ساما، مع ذكر المصدر لكل إجابة، ورفض صريح للإجابة إذا كان السؤال خارج قاعدة المعرفة (بدل التخمين).
- تفعيل زر الإعدادات (يفتح نافذة "عن النظام" ويقود لتبويب الحدود) وزر الإشعارات (يعرض آخر المعاملات المحظورة/المعلّقة) — كانا بلا وظيفة سابقاً.
- إصلاح `vite.config.js`: لم يكن يحتوي فعلياً على الـ proxy الذي يوثّقه هذا الملف لطلبات `/api`، رغم أن الواجهة تعتمد عليه بالكامل للتواصل مع الخلفية.

## البنية

المشروع بنية مسطّحة (flat) — الخلفية والواجهة في نفس المجلد الجذر، وليستا في مجلدين منفصلين:

```
(Meyar) System Dashbord/
├── main.py                 # FastAPI — بيانات حية + محرك المستويين + الشات بوت
├── package.json
├── vite.config.js          # يتضمن الآن proxy فعلي لـ /api → localhost:8000
├── tailwind.config.js
├── postcss.config.js
├── index.html
└── src/
    ├── main.jsx
    ├── index.css
    └── MeyarDashboard.jsx   # المكوّن الرئيسي الكامل
```

## تشغيل الخلفية (Backend)

```bash
pip install fastapi uvicorn pydantic
python main.py
# أو: uvicorn main:app --reload --port 8000
```

يعمل على `http://localhost:8000`، مع نقاط النهاية التالية:

- `GET /api/compliance-summary`
- `GET /api/realtime-transactions?limit=40&status=blocked`
- `GET /api/compliance-trends`
- `GET /api/regulatory-updates`
- `POST /api/chatbot/query` — body: `{"question": "...", "lang": "ar"}`
- `GET /api/chatbot/suggested-questions`

## تشغيل الواجهة الأمامية (Frontend)

```bash
npm install
npm run dev
```

يعمل على `http://localhost:5173` مع إعادة توجيه (proxy) تلقائية لكل طلبات `/api` إلى الخلفية على المنفذ `8000` (مضبوطة الآن فعلياً في `vite.config.js`).

> ملاحظة: إذا تعذّر الاتصال بالخلفية لأي سبب، تعمل الواجهة تلقائياً ببيانات محلية واقعية (fallback) — بما فيها الشات بوت ونموذج المستويين — حتى لا تظهر الشاشة فارغة أو معطّلة أثناء العرض التجريبي.

## أبرز الميزات

- **الشريط الجانبي التنفيذي**: خمس تبويبات وظيفية بالكامل (نظرة عامة، المراقبة اللحظية، التحليلات، محرك التشريعات، الحدود والمسؤولية) مع إمكانية الطي.
- **بطاقات المؤشرات (KPI)**: أربع بطاقات نيون فاخرة بمخططات صغيرة (sparklines) حية.
- **راصد القرار اللحظي**: قائمة معاملات حيّة تُحدَّث كل 6 ثوانٍ، تُصنَّف كل معاملة إلى مستوى ١ (منع آلي) أو مستوى ٢ (تعليق + مراجع بشري مُسمّى) مع فلاتر تفاعلية وبحث فوري.
- **التحليلات**: مخطط مركّب بمحورين (Recharts) يقارن الالتزام الفعلي بالهدف المستهدف مقابل نسبة خفض التكاليف التشغيلية، مع شرح المنهجية.
- **محرك التشريعات الذكي**: عرض تعاميم ساما وحالة تحويلها الآلي إلى قواعد برمجية تنفيذية.
- **مساعد التشريعات**: نافذة محادثة عائمة تجاوب من قاعدة معرفة ساما المحلية فقط.
